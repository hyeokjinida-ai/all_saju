# 명운록(命運錄) 사주 사이트 — 현재 현황 정리 (GPT 상담용)

> 이 문서는 ChatGPT 등과 상담하기 위해 **지금 실제로 만들어진 사이트의 구조·퍼널·상태**를 정리한 것입니다.
> 아이디어/발주서가 아니라 **현재 구현된 사실** 기준입니다.

---

## 1. 서비스 한 줄 정의

- **무엇**: 40~50대를 위한 사주 해석 **판매 사이트**.
- **포지셔닝**: "사주를 파는" 게 아니라, **돈·가족·부부·자녀·사업·노후·건강 같은 4050 현실 고민을 사주 언어로 풀어주는 개인 해설**을 판다.
- **수익 모델**: 무료 명식으로 유입 → ₩9,900 간명 결과지 결제 → ₩49,900 심명 업셀 / 크로스셀 재구매.
- **자동화**: 결제부터 사주 계산(만세력), AI 해석(LLM) 생성까지 백엔드 자동.

---

## 2. 기술 스택 (이미 구축됨)

- **프론트/서버**: Next.js 15 (App Router) + React 19 + Tailwind
- **DB/인증**: Supabase (Postgres + Auth)
- **결제**: 토스페이먼츠 (위젯 + 웹훅)
- **사주 계산**: 만세력 API (천간지지/십성/대운/세운/12운성/신살 등)
- **AI 해석**: LLM (유료 결과지 생성). 무료 결과는 LLM 없이 규칙 기반 계산.
- **배포 대상**: 보일러플레이트 기반(수강생 배포용 구조).

---

## 3. 상품 라인업 (실제 DB 시드 기준)

| slug | 이름 | 가격 | 역할 |
|---|---|---|---|
| today-fortune | 오늘의 운세 한 줄 | ₩4,900 | 가벼운 진입 |
| **basic-saju** | 기본 사주 풀이 (= **간명**) | **₩9,900** | **퍼널 메인 / 무료 진입점** |
| love-saju | 연애·궁합 리포트 | ₩19,900 | 크로스셀 |
| **premium-saju** | 프리미엄 종합 풀이 (= **심명**) | **₩49,900** | **업셀** |

> ⚠️ 현재 상품은 이 **4개뿐**. ("월별 흐름 ₩19,900", "가족 재물 ₩29,900" 같은 상품은 **아직 없음** — 발주서 예시일 뿐.)

## 4. 핵심 퍼널 (현재 구현된 흐름)

```
랜딩(/)
  └─ "무료로 내 명식 확인하기"
       ↓
입력 위저드 (/products/basic-saju)   ← flow="free" 모드
  이름 → 생년월일 → 태어난시각(모름 가능) → 성별 → 양력/음력 → 고민 → 입력확인
  └─ "이 정보로 무료 결과 보기"  (로그인·결제 없이 진행)
       ↓  (입력값을 sessionStorage 에 저장, /free-result 로 이동)
무료 결과 (/free-result)
  8글자 · 일간성향 · 오행균형 · 강한/부족한기운 · 올해흐름 · 고민맛보기
  └─ "내 사주 전체 흐름 자세히 보기 · 간명 ₩9,900"  (여기서 로그인 유도)
       ↓
결제 (/checkout/[orderId])   ← 토스 위젯
       ↓  (결제 성공 → 웹훅 → 만세력 계산 → LLM 해석 생성)
간명 결과지 (/results/[resultId])
  명식표 + LLM 마크다운 해석
  └─ 하단: 심명(₩49,900) 업셀 + 개인화 크로스셀(최대 3개, 재입력 없는 원클릭 재구매)
```

### 위저드 "이중 모드" (중요 설계)
- `flow="free"` (basic-saju): 결제·로그인 없이 → /free-result 로. **무료 유입 전용.**
- `flow="order"` (love-saju, premium-saju, today-fortune): 바로 주문 생성 → 결제. **직접 구매용.**
- 같은 위저드 컴포넌트를 두 모드로 재사용.

## 5. 전체 페이지(라우트) 목록 — 실제 존재

| 경로 | 화면 | 디자인 상태 |
|---|---|---|
| `/` | 랜딩 | ✅ 명운록(와인+골드) 적용 완료 |
| `/products` | 상품 목록 | △ 부분 적용 |
| `/products/[slug]` | 상품 상세 = 입력 위저드 | ✅ 적용 완료 |
| `/free-result` | 무료 결과 | ✅ 적용 완료 |
| `/checkout/[orderId]` | 결제 | ✗ 기본(밋밋) |
| `/results/[resultId]` | 유료 간명 결과지 | △ 헤더만 적용, 본문 밋밋 |
| `/checkout/success` `/checkout/fail` | 결제 성공 / 실패 | ✗ 기본 |
| `/login` `/signup` `/reset` | 로그인 / 회원가입 / 비번재설정 | ✗ 기본 |
| `/mypage` `/mypage/orders` `/mypage/reviews` | 마이페이지(주문·리뷰) | ✗ 기본 |
| `/mypage/orders/[orderId]/review` | 리뷰 작성 | ✗ 기본 |
| `/admin` `/admin/login` `/admin/orders` | 관리자(내부용) | ✗ 기본 |
| `/legal/terms` `/legal/privacy` `/legal/refund-policy` | 약관/개인정보/환불 | ✗ 기본 |
| `/demo` | DB 없이 명식→결과지 미리보기(개발용) | △ |

### API 라우트 (백엔드, 이미 작동 — 실제 파일 기준)
- `POST /api/orders/create` — 주문 생성 (위저드·크로스셀 공용)
- `POST /api/orders/confirm` — 결제 승인
- `POST /api/saju/preview` — 무료 결과용 명식+규칙기반 해석
- `POST /api/generate-manseryeok` — 만세력 계산 + (유료) LLM 결과지 생성
- `POST /api/reviews` — 리뷰 작성
- `POST /api/auth/signout` — 로그아웃
- `GET /auth/callback` — Supabase 인증 콜백

```
재물 · 부부·연애 · 자녀 · 직장·사업 · 건강 · 올해 운 · 노후 · 가족
```
(이 글자가 정확히 일치해야 시스템이 작동. 발주서엔 6개였지만 실제는 8개.)

## 7. 디자인 정체성 (명운록)

- **무드**: 한지·먹·인장(印章) + 시네마틱 야경. **딥 와인/블랙 배경 + 골드(금박) 포인트.**
- **색**: 와인 #3a0d18 / 골드 #d4af6a / 본(미색) #f0e6d2 / 인장(붉은) #8b1e1e / 오행 5색.
- **폰트**: 명조(Nanum Myeongjo/Noto Serif KR), 붓글씨 한자(Ma Shan Zheng), 숫자(JetBrains Mono).
- **규칙**:
  1. 한자(命運錄/간명/심명/推)는 **장식·브랜딩 전용**. 버튼·가격은 **쉬운 한국어**.
  2. ₩9,900(간명)과 ₩49,900(심명)을 **나란히 노출 금지**. 심명은 뒤에서만 업셀.
  3. 개인정보 문구 필수: "입력하신 정보는 명식 계산과 결과 생성에만 사용됩니다."
  4. 4050 가독성: 큰 글씨·명확한 버튼·충분한 여백.

## 8. 지금까지 완료된 것 vs 남은 것

### ✅ 완료
- 무료 유입 퍼널 (랜딩 → 위저드 → 무료결과 → 간명 결제 유도) 전 구간 구현·검증
- 랜딩 4050 리포지셔닝(현실고민 hero, 무료 우선, CTA 다양화, 안심 박스)
- 위저드(모름 옵션, 양·음력 안내, 입력확인)
- 무료 결과(8글자·일간·오행·올해흐름·고민맛보기 실제 티저)
- 크로스셀(고민 기반 정렬, 최대 3개)
- 간명 결과지 LLM 프롬프트 톤(현실언어·반바넘·고민연결)

### ✗ 남은 것 (주로 디자인)
- 유료 결과지(/results) 본문 디자인 — "만원 값 하는" 고급 리포트 느낌
- 결제(/checkout) 디자인
- 마이페이지·로그인·결제성공/실패 디자인
- (선택) 크로스셀용 신상품 추가 여부 결정
- (선택) 결과지 PDF저장/공유 기능
