# 자체 만세력 전환 로드맵

> 외부 만세력 API(luckyloveme) 의존을 졸업하고, 자체 계산 엔진으로 전환하기 위한 계획.
> 2026-06-03 실제 검증 결과 기반. 결론: **명식·십성·오행·대운방향은 무료로 즉시 재현 가능(검증 완료).**

---

## 1. 왜 전환하나

| 항목 | 외부 API (현재) | 자체 엔진 |
|---|---|---|
| 운영 비용 | 건당 과금 가능 / 체험키 6,000회 한도 | **0원·무제한** |
| 종속성 | API사 가격인상·종료 시 사업 멈춤 | 안전 |
| 한도/장애 | 분당 제한 | 없음 |

→ 손님이 월 수천 건 넘으면 전환 가치가 커짐. 그 전엔 외부 API로 충분.

## 3. 사용 도구

### 검증된 오픈소스
- **`@fullstackfamily/manseryeok`** (npm, v1.0.8) — KASI(한국천문연구원) 데이터 기반. 명식·절기·음양력 제공. 1000~2050년. DB 불필요.
  - 제공: `calculateSaju(y,m,d,h,min)` → 명식 4기둥 + 진태양시 보정
  - 한계: 절기 **이름**만 주고 **정확한 절입 날짜·시각 없음** → 대운수 계산 불가
- ⚠️ `manseryeok`(yhj1024) 패키지는 `workspace:` 의존성 오류로 설치 안 됨 — 사용 불가

### 자체 계산 코드 (오늘 작성, 동작 확인됨)
- 십성: 일간 오행·음양 vs 대상 오행·음양 → 비겁/식상/재성/관성/인성 (`_sipseong_test.cjs`)
- 대운 방향: 년간 음양 × 성별 → 순행/역행
- 대운 간지: 월주에서 60갑자 순/역 전개

## 4. 남은 작업과 해결법

| 항목 | 방법 | 난이도 | 예상 |
|---|---|---|---|
| **대운수(절입시각)** | 1900~2050 절입시각 테이블 확보(또는 `astronomia`로 태양황경 계산) → "생일~절기 일수÷3" 공식(이미 작성됨)에 연결 | ★★ | 0.5일 |
| **12운성** | 일간×지지 120칸 고정표 박기 | ★ | 0.5일 |
| **신살** | 공식 if문 (강의 3주차 slide 77에 도화살 등 공식 있음) | ★★ | 2~3일 |
| **격국** | 월지 십성 매핑 (십성 이미 계산됨 → 거의 완성) | ★ | 0.5일 |
| **검증 자동화** | 무료 라이브러리 교차검증 또는 축적된 raw_analysis와 자동 대조 | ★★ | 1~2일 |
| **합계** | | | **약 1~2주** |

→ 전부 "정해진 표/공식"이라 창작 아님. 노가다는 **코드(AI)가 자동 처리**, 사용자는 방향 결정만.

## 5. 검증 비용 0회 전략 (체험키 안 쓰고 검증)

luckyloveme 호출 없이 검증하는 법:
1. **축적된 raw_analysis 재사용** — 0005 마이그레이션으로 손님 결과의 16종 원본을 DB에 저장 중. 이게 정답 데이터셋.
2. **무료 라이브러리 교차검증** — 설치되는 무료 만세력 2개를 서로 대조 (둘 다 KASI 기반이면 정답)
3. **표준 공식** — 명식·십성·대운은 천문/명리 공식이라 정답이 하나. luckyloveme 불필요.

→ 명식류는 체험키 **0회**로 검증 가능. 신살만 "luckyloveme와 똑같이 맞추려면" 소량 필요.

## 6. 데이터 축적 전략 (이미 가동)

- **0005_raw_analysis 마이그레이션** 적용됨 → confirm 라우트가 손님마다 luckyloveme 16종 원본을 `saju_results.raw_analysis`에 저장.
- 손님 받을수록 정답 데이터셋이 쌓임 → 나중에 자체 엔진 검증·신살 복제에 사용.
- ⚠️ **DB(Supabase) 연결 환경에서만 저장됨.** 마이그레이션 `supabase/migrations/0005_raw_analysis.sql` (또는 `_apply_all.sql`)을 Supabase에 적용해야 함.

## 7. 단계별 전환 계획

| 시점 | 만세력 | 작업 |
|---|---|---|
| **지금~런칭** | 외부 API | 그대로. 체험키 6,000회로 초기 판매·데이터 축적 |
| **월 수백 건** | 명식·십성·오행·대운 자체 + 나머지 API | `@fullstackfamily/manseryeok` + 자체 계산 도입 (비용 큰 부분부터) |
| **월 수천 건** | 완전 자체 | 절입시각 테이블 + 신살·격국 룰 + 검증 → API 졸업 |

## 8. 법적 주의

- **명식·십성·대운 = 천문/명리 공식 (저작권 없음)** → 자체 계산 자유.
- ⚠️ luckyloveme의 **해석 '문구'를 복제하는 것**은 약관 위반 가능. 우리는 AI가 해석을 새로 쓰므로 문제없음.
- raw_analysis 저장은 '내 손님 결과 보관'이라 일반적으로 OK. 단, 그걸로 동일 API를 상품화하진 말 것.
