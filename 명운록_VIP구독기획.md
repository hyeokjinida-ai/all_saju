# 명운록 — VIP & 구독 기획

> 목적: 49,900 위에 **앵커(VIP)** 를 세워 종합 전환을 끌어올리고, **구독**으로 반복매출(LTV)을 만든다.

---

## 1. 인생 VIP 정밀 풀이 — ₩99,000 (구현 완료, 비활성)

### 무엇이 다른가 (premium 49,900 대비)
premium은 10챕터, VIP는 **13챕터**로 더 깊다. 검증(2026-06-04, /demo?slug=vip-saju)에서 13챕터 정상 생성 확인.

| 추가되는 차별 챕터 | 내용 |
|---|---|
| 대운 60년 연표 | 10년 단위로 인생의 큰 계절을 통째로 정리 |
| ★향후 10년 집중 로드맵 | 해마다 무엇을 하고 무엇을 피할지 |
| 결정적 시기 3가지 | 이직·투자·이사·인연의 "황금 타이밍" 콕 |
| 나에게 맞는 것/피할 것 | 사람·직업·환경·방향 |

### 가격 전략에서의 역할
- **디코이/앵커**: 99,000이 존재하면 49,900이 "합리적 중간 선택"이 된다 → 종합 판매↑ (안 팔려도 효과).
- 진짜로 사는 0.5~2%는 객단가를 통째로 끌어올린다.

### 추가하면 더 좋을 실물 가치(선택)
- **PDF 증서 다운로드**(평생소장) — 원가 0, 체감가치 큼.
- **1년 무제한 추가질문** — 이미 결과지 본 사람의 재접점.

### 활성화 방법 (사장님)
현재 `is_active: false`(준비됨, 고객 노출 X). 켜는 법:
- `src/config/products.seed.ts` 에서 vip-saju 의 `is_active: true` → `npm run seed:products`, 또는
- 관리자/DB에서 products.vip-saju 의 is_active 를 true 로.
- 켜면 /products·상세·결제까지 자동 노출(검증된 13챕터 생성).

> 권장: **종합(49,900) 다음으로 결과지 크로스셀에서만 슬쩍 노출**하거나, 상품목록 최상단 "프리미엄" 자리에. 입구(무료)에서 9,900과 직접 병렬은 피한다(가격 충격).

## 2. 구독 — 반복매출(LTV) 설계

### 상품안
| 상품 | 가격 | 내용 |
|---|---|---|
| 월 운세 구독 | 월 9,900 / 연 79,000 | 매월 1일 "이달의 운 리포트" + 오늘의 운세 무제한 |
| (옵션) 가족 구독 | 연 129,000 | 본인+가족 2~3인 |

### 왜 구독인가
- 사주는 **계절(세운·월운)이 매달/매년 바뀐다** → 재구매 명분이 자연스럽다.
- 단건 판매는 1회성, 구독은 LTV가 몇 배. 카카오 채널 알림톡과 결합하면 이탈도 적다.

### 구현 난이도 (정직하게)
단건 결제와 달리 **정기결제(빌링키) 연동이 필요**하다 → 현재 구조(토스 단건 위젯)에 추가 개발 필요:
1. 토스 **billingAuth**(카드 등록) → billingKey 발급/저장(암호화).
2. 매월 크론(스케줄러)으로 billingKey 자동 청구.
3. 구독 상태 테이블(subscriptions), 해지/실패/재시도 처리.
4. 마이페이지 구독 관리 UI.

> 추정: 단건 결제보다 2~3배 작업량. **먼저 "연 이용권"을 단건 상품(연 79,000, 1회 결제로 1년 콘텐츠 제공)으로 출시**해 수요를 검증한 뒤, 반응이 좋으면 진짜 정기결제로 가는 것을 권장. (연 이용권은 지금 구조로 바로 가능)

---

## 3. 추천 출시 순서

1. **VIP 활성화**(즉시 가능, 검증됨) → 앵커 효과로 종합 판매↑
2. **연 이용권 단건**(79,000, 빠르게) → 구독 수요 검증
3. 반응 좋으면 **정기결제 구독** 본구현
