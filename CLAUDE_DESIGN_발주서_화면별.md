# 명운록(命運錄) 사주 사이트 — 디자인 발주서 (화면별)

> **이 문서의 용도**: claude.ai/design 에 그대로 붙여넣어, 아래 화면들의 **디자인 시안 + Claude Code용 핸드오프(HTML/Tailwind)** 를 받기 위한 발주서입니다.
> 실제 구현 코드(Next.js)는 Claude Code가 맡습니다. claude.ai/design은 **이미 정해진 디자인 토큰 안에서** 화면 레이아웃·시안만 그려주면 됩니다.

---

## 0. 프로젝트 한 줄 요약

- **무엇**: 40~50대를 위한 사주 해석 판매 사이트. "사주를 파는" 게 아니라 **돈·가족·관계·일/사업·노후·건강 같은 현실 고민을 사주로 풀어주는** 서비스.
- **분위기**: 명운록(命運錄) — 한지·먹·인장(印章) 무드 + 시네마틱 야경(딥 와인/블랙 + 골드). 고전적이고 신비롭되, 40~50대가 읽기 편하게.
- **스택**: Next.js 15 (App Router) + Tailwind + 기존 CSS 토큰. 결제는 토스페이먼츠.

## 2. 절대 규칙 (디자인이 어겨선 안 됨)

1. **한자(命運錄/簡命/深命/推/命式 등)는 장식·브랜딩 전용.** 버튼·가격·실제 기능 텍스트는 **쉬운 한국어**로. 예: 버튼은 "간명 결과지 보기 9,900원" (O), "簡命 9,900" (X).
2. **가격 노출 규칙**: 메인 결제 지점에서는 **₩9,900(간명)만** 보여줌. **₩49,900(심명)은 절대 ₩9,900과 나란히 놓지 않음** — 무료결과 하단/결과지 중간·하단/크로스셀에서만 업셀로 등장.
3. **개인정보 안심 문구**는 입력·결과 화면에 반드시: "입력하신 정보는 명식 계산과 결과 생성에만 사용됩니다."
4. **40~50대 가독성**: 본문 최소 15~16px, 충분한 줄간격, 버튼 크게, 대비 충분히.
5. 와인+골드 다크 톤이 기본. 단, **결과지·증서류는 밝은 한지(`.hanji`) 카드**로 대비를 줘도 좋음(아래 명시).

## 4. 화면별 상세 (시안 발주 내용)

### ★★★ 1. 유료 결과지 `/results/[resultId]` — **가장 중요**
돈을 낸 고객이 처음 보는 "결과물". 여기 만족도가 재구매·후기·환불을 좌우함.

- **데이터**: 상품명, 생년월일·성별, 그리고 LLM이 만든 **마크다운 본문**(## 헤딩 + 문단 + 불릿).
- **현재**: 마크다운을 `.prose-saju`로 흘려보내기만 함 → 그냥 글 덩어리. 밋밋.
- **원하는 디자인** (claude.ai/design에서 만든 폰 목업 방향이 정답):
  - 상단: **증서/기록(命運錄) 느낌의 헤더** — 제목 + 받는 사람 정보 + 발행일 + 붉은 낙관(`.seal` 命錄).
  - 본문: 마크다운 `##` 섹션을 **"PART 1·2·3·4" 처럼 구획된 카드/블록**으로. 각 섹션 제목에 한자 장식 + 골드 디바이더(`.gold-rule`).
  - 밝은 **한지(`.hanji`) 결과지 카드**를 와인 배경 위에 띄우는 대비 구성 추천(목업처럼).
  - 핵심 문장은 강조(골드 또는 밑줄), 조언은 별도 박스.
  - 하단: **업셀(심명 ₩49,900) + 크로스셀(최대 3개)** 영역.
  - "PDF로 저장/카톡 공유" 같은 액션 버튼 자리(기능은 Claude Code가).
- **상태 디자인 2개 필요**:
  1. 정상 결과
  2. "결과지 준비 중…"(결제는 됐지만 생성 대기) — 명식을 세우는 듯한 로딩 연출.

### ★★★ 2. 결제 `/checkout/[orderId]`
- **데이터**: 상품명, 금액, 토스페이먼츠 위젯(외부 iframe 영역).
- **원하는 디자인**: 주문 요약 카드(상품·금액) + 안심 문구(보안 결제, 입력정보 사용 범위) + 토스 위젯이 들어갈 깔끔한 컨테이너. 골드 프레임/낙관으로 "정식 발주" 느낌.
- 모바일 우선. 버튼 크게.

### ★★ 3. 결제 성공/실패 `/checkout/success`, `/checkout/fail`
- 성공: 축하 + "결과지 보러가기" 큰 CTA + 다음 단계 안내. 낙관/별밤(`.scene-cosmos`) 연출 OK.
- 실패: 침착한 안내 + "다시 시도/문의" + 결제 안 됨 명시.

### ★★ 4. 마이페이지 `/mypage`, `/mypage/orders`, `/mypage/reviews`
- **주문 목록**: 카드/리스트로 상품명·날짜·상태(결제완료/준비중)·"결과 보기". 빈 상태 디자인 필요.
- **리뷰**: 별점 + 내용. 작성 폼 + 내가 쓴 리뷰 목록.
- 4050이 한눈에 알아보게 — 상태 배지 색·라벨 명확히.

### ★★ 5. 인증 `/login` `/signup` `/reset`
- 카카오 로그인 버튼(`.btn-kakao`)을 1순위로 크게. 이메일은 보조.
- 명운록 톤의 미니멀 폼. 안심 문구. (계정 생성·비번 입력은 사용자가 직접 — 디자인만)

### ★ 6. 상품 목록 `/products`
- today-fortune ₩4,900 / basic-saju ₩9,900(무료 진입 강조) / love-saju ₩19,900 / premium-saju ₩49,900.
- 카드 그리드. **basic-saju는 "무료로 시작" 강조**, premium은 "더 깊게" 톤. (가격 나란히 규칙은 메인 퍼널에만 적용 — 목록 페이지는 전 상품 노출 OK)

### ★ 7. 약관/개인정보/환불 `/legal/*`
- 긴 텍스트 가독성만. 명조 본문, 적당한 폭(max-w), 목차/소제목.

## 5. 참고용 — 이미 완성된 화면 (디자인 일관성 맞추기용, 수정 금지)
- `/` 랜딩: hero(현실고민) → 이런분께 → 무료로 확인 → 유료샘플 → 후기/신뢰 → 가격(무료 우선) → 최종CTA + 스티키바.
- `/products/basic-saju`: 무료 명식 입력 위저드(이름→생일→시각→성별→달력→고민→확인). `.ap-input` 사용.
- `/free-result`: 8글자·일간·오행 막대·올해 흐름·고민 맛보기 → 간명 결제 CTA → 심명 보조링크.
→ 위 톤(여백, 골드 디바이더, 낙관, 명조)을 **결과지/결제/마이페이지에도 그대로 확장**해주세요.

## 6. claude.ai/design에 부탁하는 산출물 형식

각 화면마다:
1. **HTML + Tailwind(또는 inline style) 시안** — 위 토큰/유틸 클래스를 쓴 정적 프로토타입.
2. 모바일(우선) + 데스크탑 2뷰.
3. 가능하면 **Claude Code용 핸드오프**: 화면별 파일 + "이 블록은 어떤 데이터/컴포넌트에 매핑되는지" 메모.
4. 위 **절대 규칙(2번)** 준수 여부 체크리스트.

> 데이터 구조·라우팅·결제 연동은 Claude Code가 처리하니, claude.ai/design은 **레이아웃과 시각 시안에 집중**해주면 됩니다.
