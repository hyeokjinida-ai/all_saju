# 명운록 — 퍼널 설계 (한 번 들어오면 못 나가게)

> 목표: 방문 → 무료 체험 → 첫 결제 → 추가 결제 → 재방문까지, **각 단계의 이탈(누수)을 막고** 객단가·재구매를 끌어올린다.
> 원칙: **마찰은 줄이고(쉽게), 갈증은 키우고(궁금하게), 떠날 때 한 번 더 잡는다(회수).**

---

## 0. 전체 퍼널 한눈에

```
광고/유입
   │  (후킹 카피로 클릭)
   ▼
랜딩 ─ "지금 그 답답함엔 분명한 이유가 있습니다"
   │  CTA 단 하나: 무료로 내 명식 확인  ← 선택지 최소화
   ▼
무료 위저드(7단계) ─ 한 화면 한 질문, 진행바로 완주 유도
   │  입력은 sessionStorage 저장(이탈해도 보존)
   ▼
무료 결과 ─ 명식 8글자 + 첫 문장 보여주고 ★나머지는 블러(갈증 끊기)
   │  "여기까지가 무료, 진짜 답은 여기서"
   ▼
첫 결제 9,900 ─ 카카오 3초 로그인(마찰 0) → 토스 결제
   │
   ▼
결과지 ─ 그 사람 약점 짚으며 ★개인화 크로스셀
   ├─ 재물 19,900 / 월별 24,900 / 종합 49,900
   ▼
재방문 ─ 오늘의 운세·월별 리포트·구독 → LTV
```

---

## 1. 단계별 "이탈 차단" 장치

### ① 유입 → 랜딩 (이탈 원인: 안 끌림)
- ✅ **후킹 헤드라인**(통증 직격) + CTA **단 하나**(무료). 선택지가 많으면 이탈한다.
- ✅ 사회적 증거(누적 11,300명·★4.96)·권위(천년 계보·87.2%)를 스크롤 곳곳에.
- 추가: **퇴장 의도(exit-intent)** — 마우스가 상단으로 나갈 때 "떠나기 전, 무료 명식만 보고 가세요" 1회 노출.

### ② 위저드 (이탈 원인: 입력 귀찮음·이탈 시 유실)
- ✅ 한 화면 한 질문 + 진행바(완주 심리). 성별/달력은 탭 즉시 자동 진행.
- ✅ **입력 자동 저장**: 무료(SAJU_INPUT_KEY)·주문(ORDER_DRAFT_KEY) 모두 sessionStorage 보존 → 새로고침/로그인 왕복에도 안 날아감.
- 추가: 이탈 후 재방문 시 "이어서 입력하시겠어요?" 복귀 배너.

### ③ 무료 결과 → 첫 결제 (이탈 원인: 결제 마찰·동기 부족) ★최대 누수
- ✅ **갈증 끊기**: 첫 문장만 보여주고 나머지 블러. "내 얘기네" → 더 보고 싶게.
- ✅ **카카오 3초 로그인**(신규) — 기존엔 이메일 로그인 강제라 여기서 대거 이탈했음. 원탭으로 마찰 제거.
- ✅ **입력 보존 복원** — 로그인 다녀와도 확인 단계로 복원돼 바로 결제.
- 추가: 결제 직전 **후기·환불보장** 문구로 안심. "안 맞으면 100% 환불".

### ④ 첫 결제 → 추가 결제 (객단가) ★ROI 최고 구간
- **결과지 하단 개인화 크로스셀**(CrossSell 컴포넌트 활용): 그 명식에서 가장 약한/이슈 영역을 짚으며 추천.
  - 예: "재성이 0개 — 돈이 머물 구조가 약합니다 → [재물 깊이보기]"
- 신규 유입보다 **이미 결제한 사람에게 30초 안에** 파는 게 10배 싸다.
- 묶음 유도: "심화 3종을 한 번에" → 종합 49,900 업셀.

### ⑤ 재방문 (이탈 원인: 다시 올 이유 없음)
- **월별 캘린더·오늘의 운세** = 자연스러운 재방문 트리거.
- **구독(연 79,000)**: 매월 운 리포트 → 반복 매출.
- 카카오 채널 친구추가 → 알림톡으로 "이번 달 운세 나왔어요" 회수.

## 2. 회수(미결제 이탈자 다시 끌어오기)

- **장바구니/주문 미완료 회수**: pending 주문이 N시간 방치되면 알림톡/메일 "결제만 하면 결과지가 나옵니다".
- **카카오 알림톡**(채널 연동 시): 가장 저렴하고 열람률 높은 회수 채널.
- **리타겟팅 픽셀**(메타/구글): 방문·이탈자에게 광고 재노출.

## 4. 카카오 로그인 — 작동시키려면 (관리자 설정)

코드는 다 됐습니다. **실제 작동하려면 아래 2곳 설정이 필요**합니다(코드로는 불가, 콘솔 작업).

### A. 카카오 개발자 콘솔 (developers.kakao.com)
1. 애플리케이션 생성 → **REST API 키** 확보.
2. [카카오 로그인] 활성화 ON.
3. **Redirect URI** 등록: `https://<프로젝트>.supabase.co/auth/v1/callback`
4. [동의 항목]에서 **닉네임/이메일** 등 필요한 항목 동의 설정.

### B. Supabase 대시보드
1. Authentication → Providers → **Kakao** 활성화(ON).
2. 카카오의 **REST API 키**(Client ID) + (필요 시) Client Secret 입력.
3. Authentication → URL Configuration → **Redirect URLs** 에 사이트 콜백 추가:
   - `http://localhost:3000/auth/callback` (개발)
   - `https://<운영도메인>/auth/callback` (운영)

> 설정 전에는 버튼 클릭 시 "아직 설정되지 않았어요" 안내가 뜨거나 Supabase 오류 페이지로 갑니다. 설정 후엔 카카오 동의 → `/auth/callback` → 원래 결제 지점으로 자동 복귀합니다.

### 코드 동작 방식(참고)
- `KakaoLoginButton` → `supabase.auth.signInWithOAuth({ provider:"kakao", redirectTo:"/auth/callback?next=<목적지>" })`
- `/auth/callback` → `exchangeCodeForSession` 후 `next` 로 이동.
- 위저드에서는 카카오 이동 직전 `saveDraft()` 로 입력 저장 → 복귀 후 복원.
