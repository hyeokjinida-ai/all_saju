# 명운록(命運錄) 사주 서비스 — Claude Design 발주서

> 이 문서를 Claude Design(claude.ai/design)에 그대로 붙여넣어 화면 디자인을 요청하세요.
> **반드시 지켜야 할 제약**과 **자유롭게 디자인할 영역**을 구분해 두었습니다.

## 0. 한 줄 요약

정통 만세력 + AI가 풀어주는 한국식 사주 분석 리포트를 **온라인에서 결제하고 즉시 받아보는** 서비스.
디자인 컨셉은 **명운록(命運錄) — "운명을 기록하다"**, 다크 와인 + 골드 + 한국 명조체의 고급스럽고 신비로운 톤.

---

## 1. ⚠️ 반드시 지켜야 할 기술 제약 (디자인이 깨면 안 되는 것)

이 서비스는 이미 작동하는 백엔드(결제·사주분석·AI)가 붙어 있습니다. 디자인은 **아래 흐름과 데이터를 바꾸지 않는 선에서** 만들어야 합니다.

### 1-1. 전환 퍼널 (이 순서 고정)
```
① 홈(랜딩)
   ↓
② 상품 목록  →  ③ 상품 상세 (+ 사주 입력 폼)
   ↓ "결제하러 가기"
④ 토스페이먼츠 결제 위젯 (토스가 그리는 iframe — 디자인 변경 불가)
   ↓ 결제 성공
⑤ 결과지 (사주 명식표 + AI가 쓴 마크다운 해석)
```

### 1-2. 사주 입력 폼 — 필드 스키마 (이 항목/타입 고정)
| 필드 | 타입 | 필수 | 비고 |
|---|---|---|---|
| 이름 | text | 선택 | placeholder "홍길동" |
| 생년월일 | date | **필수** | |
| 출생 시각 | time | 선택 | "시 모름" 체크박스와 연동(체크 시 비활성) |
| 성별 | 선택 | 필수 | 남성 / 여성 (2지선다) |
| 달력 | 선택 | 필수 | 양력 / 음력 (2지선다) |
| 고민 | 복수선택 | 선택 | 연애 · 결혼 · 직장 · 재물 · 건강 · 학업 · 이직 · 사업 (칩 형태) |

> CTA: 로그인 상태면 **"결제하러 가기"**, 비로그인이면 **"로그인하고 결제하기"** (분기 유지)

### 1-3. 결과지 구성 (고정 데이터)
- **사주 명식표**: 시주 / 일주 / 월주 / 년주 4기둥 × (천간/지지) 표
- **AI 해석 본문**: 마크다운(제목·문단·리스트·표 포함)으로 렌더링됨 → 긴 글이 예쁘게 보이는 타이포그래피 필요
- 상단 메타: 상품명, LLM 제공자/모델, 생성일시

### 1-4. 기술 스택 (참고)
- Next.js 15 (App Router) + Tailwind CSS + 토스페이먼츠 위젯 + Supabase
- 즉, **Tailwind 클래스 / React 컴포넌트로 바로 옮길 수 있는** 디자인이어야 함

```
/* 오행(五行) 색 — 명식/요소 표현용 */
```

### 폰트
- **제목 / 한글 본문**: `Nanum Myeongjo`, `Noto Serif KR` (명조체)
- **한자 장식(붓글씨)**: `Ma Shan Zheng` — 命運錄, 命式 같은 한자 라벨에 사용
- **숫자/가격/메타**: `JetBrains Mono`
- 자간을 살짝 넓게(tracking 0.04em~) 써서 고급스러운 느낌

### 톤 & 무드
- 다크 와인 배경 위 금빛 강조, 코스모스(별·우주) 느낌의 그라데이션

- 버튼: 금빛 필(pill) 버튼 + 와인색 글자

## 3. 📐 기운사(경쟁사) 벤치마크에서 가져올 2가지

> 디자인 레퍼런스: https://giunsa.com (소프트한 밝은 톤 → **우리는 명운록 다크 와인으로 재해석**)

### (A) 챕터형 "결과 리포트 미리보기" — 전환율 핵심 장치
기운사는 상세 페이지에서 **결과지에 어떤 내용이 담기는지** 챕터별로 길게 미리 보여주며 구매를 유도합니다.

**우리 상품 상세 페이지에 추가할 것:**
- "이 리포트에는 이런 내용이 담깁니다" 섹션
- 챕터 카드(예시): `1장 타고난 성향·재능` / `2장 재물운·금전 흐름` / `3장 연애·결혼운` / `4장 직업·적성` / `5장 건강` / `에필로그 인생의 큰 흐름`
- 각 챕터는 **블러 처리된 미리보기** 또는 한두 줄 티저만 보이고 "결제 후 전체 공개" 느낌
- 명운록 톤: 두루마리/장(章) 넘기는 듯한 연출, 골드 챕터 번호(一·二·三 한자도 가능)

### (B) 멀티스텝 폼 위저드 + 신뢰 요소
- **폼을 한 번에 다 보여주지 말고 단계별 위저드**로 (예: 1단계 생년월일 → 2단계 시각/성별/달력 → 3단계 고민 선택 → 확인)
  - 진행 표시(스텝 인디케이터, 예: ●─●─○)
  - 단, **수집 필드는 1-2 스키마 그대로** (디자인만 단계로 쪼개기)
- **신뢰 요소(소셜 프루프)**:
  - 별점 5점 후기 카드 ("○○님이 구매했어요", 최근 날짜, 한두 줄 진솔한 후기)
  - 누적 구매 수 / "지금 ○명이 보는 중" 류
  - 후기 데이터는 이미 DB(reviews 테이블)에 있음 → 디자인만

## 5. ✍️ 카피라이팅 톤 (PG 심사 통과를 위한 표현 가이드)

사주 카테고리는 결제대행사(PG) 심사가 까다롭습니다. **단정적·미신적 표현 금지**:

| ✗ 피할 표현 | ✓ 권장 표현 |
|---|---|
| 100% 적중, 미래 예언 | 시기별 흐름 참고, 성향 분석 |
| 점/점술/운명 풀이 | 사주 분석, 성향 리포트 |
| 부적·굿 | (취급 안 함) |

- 감성 헤드라인은 OK ("운명을 기록하다", "당신을 이해하는 시간") 하지만 **결과 보장·단정은 금지**
- 상품 설명 어딘가 면책 1줄: "본 서비스는 참고용 정보이며, 의학적·법률적·재정적 결정의 근거가 아닙니다."
- 푸터에 사업자 정보(상호·대표·사업자번호·통신판매번호·주소·연락처) 노출 영역 필요

## 8. 📜 부록 A — 상품 상세 페이지(상페) 정밀 스펙 ★최우선★

> 레퍼런스: 우리가 먼저 만든 명운록 데모(https://sajuhalf-cw7z.vercel.app/).
> **한자 숫자(一~十二)를 섹션 마커로 쓰는 긴 세로 스크롤 세일즈 페이지**가 핵심 골격입니다.
> 아래 구조를 그대로 따르되, **CTA·진행·가격은 우리 자동결제 흐름**(카톡 상담 ❌ → 사주폼+토스 ⭕)에 맞춥니다.

### ⚠️ 레퍼런스와 우리의 결정적 차이 (반드시 반영)
| | 레퍼런스(sajuhalf) | 우리(명운록 보일러플레이트) |
|---|---|---|
| 결과 전달 | 카카오톡 상담 → 사람이 PDF 작성 | **사주폼 입력 → 토스 결제 → AI가 즉시 결과지 생성** |
| CTA 목적지 | `/apply` (카톡) | **상품 상세 내 사주 입력 폼 → `/checkout`** |
| 진행 단계(八行) | ①카톡 문의 ②정보 전달 ③PDF 수령 | **①상품 선택 ②사주 정보 입력 ③토스 결제 ④AI 결과지 즉시 확인** |
| 가격(十一受) | 簡命/深命 2티어 | **우리 DB 상품 가격 그대로** (단, 정가 취소선+할인 연출은 차용 OK) |
| 응답 시간 | "24h/48h 이내" | **"결제 즉시 / 수 분 내 자동 생성"** |

### 섹션 구조 (위 → 아래, 한자 마커 유지)

```
[헤더]  명運錄 命運錄 로고 + 상단 미니 카피 "천 년 명리의 줄기 끝에서"

一. Hero
   - 헤드라인: "타고난 흐름이 어디로 가는지 읽어드립니다" (감성, 단정 금지)
   - 서브: "음양오행과 십성(十星)의 흐름으로, 여덟 글자에서 성격·관계·재물·애정의 결을 정리합니다"
   - 신뢰 스탯: "누적 N건" (숫자는 데이터 바인딩)
   - 1차 CTA(부드러운): "내 사주 보러 가기" → 아래 폼으로 스크롤

二生·八字  사주가 무엇인지
   - 年·月·日·時 / 天干 4 + 地支 4 → 여덟 글자 좌표로 변환된다는 시각 설명

三異  왜 다른가 (차별화)
   - "생일만으로 60개 답안을 돌려주는 자동 앱"  vs  "당신의 여덟 글자엔 같은 답이 없습니다"
   - ※ 우리도 AI 자동이지만 "개인 명식 기반 맞춤 생성"임을 강조 (템플릿 아님)

四惑  어떤 고민에 답하나
   - 業(직업)·緣(관계)·財(재물)·時(시기) 4카드, 한자 + 한 줄 설명
   - → 폼의 '고민 복수선택' 항목과 시각적으로 연결되면 좋음

五命  분석 프레임 미리보기
   - 샘플 명식 표(예: 丁酉 己巳 己未 癸酉) + 해석 톤 맛보기

六精  방법론(신뢰)
   - 만세력 · 진태양시 · 절기 보정 · 子時 경계 처리
   - 정확도 지표 카드 (예: 명식 계산 99.7% 등 — 과장 금지, "계산 정밀도" 맥락으로)

七章  결과 리포트에 담기는 내용 ★전환 핵심★
   - 챕터 카드(번호는 한자 一二三… 또는 골드 숫자):
     1 성격의 결  2 관계의 반복  3 재물의 흐름  4 애정의 온도  5 다음 선택의 방향
   - 각 카드 일부는 **블러/티저**로만 노출 → "결제 후 전체 공개" 심리

八行  진행 방법 (★우리 흐름으로 교체★)
   - ① 상품 선택  ② 사주 정보 입력(생년월일·시각·성별·양음력·고민)  ③ 토스로 안전 결제  ④ AI 맞춤 리포트 즉시 확인
   - "사람이 아닌 정통 만세력 엔진 + AI가 즉시 작성" 강조

九覽  샘플 리포트
```

## 10. 🪜 부록 C — 단계별 사주 입력 위저드 (화면별 명세) ★우선 구현★

> 사용자가 원하는 흐름: **CTA 클릭 → 생년월일 등 정보를 한 항목씩 입력 → "다음" → 다음 단계 → … → 결제.**
> 한 화면에 한 가지(또는 한 묶음)만 물어보는 **풀스크린 스텝 방식**. 각 단계가 페이드로 전환됩니다.

### 진입
- 상페/홈의 CTA(예: **"사주 받기"**, **"내 사주 보러 가기"**) 클릭 → 위저드 시작
- 위저드는 별도 라우트(예: `/products/[slug]` 내 단계 전환) 또는 풀스크린 오버레이 — **디자인은 풀스크린 단계 화면**으로

### 공통 레이아웃 (모든 단계 동일 틀)
```
[상단]  ◀ 이전(아이콘)      진행률 인디케이터 ●─●─●─○─○─○      N/6
[중앙]  한자 라벨(작게, 골드 붓글씨)  +  질문 제목(명조, 본색)
        └ 입력 컨트롤(이 단계의 항목 하나)
        └ 도움말 한 줄(bone-soft, 작게)
[하단]  [ 다음 → ]  (골드 풀 버튼, 폭 넓게)   ※ 선택 항목은 [ 건너뛰기 ] 보조 링크
[배경]  다크 와인 + 은은한 골드 글로우 / 중앙 정렬 / 모바일 우선
```
- **전환 애니메이션**: 다음 누르면 현재 단계 살짝 위로 fade-out, 다음 단계 아래에서 fade-up
- **Enter 키**로도 "다음" 진행, 모바일에선 큰 탭 영역
- 진행률 막대 또는 점(●○) — 현재 단계 골드, 지난 단계 채움, 남은 단계 비움

### 단계 구성 (총 6단계 + 확인) — 필드/타입 고정(발주서 1-2 참조)
```
STEP 1 · 名  이름 (선택)
   질문: "어떻게 불러드릴까요?"
   입력: 텍스트 1개 (placeholder "홍길동")
   버튼: [다음 →]  +  [건너뛰기]   ← 선택 항목

STEP 2 · 生  생년월일 (필수) ★
   질문: "언제 태어나셨나요?"
   입력: 한국식 날짜 피커(연 빠른 점프). 모바일 휠/네이티브 date
   검증: 비어있으면 다음 비활성 + "생년월일을 입력해 주세요"
   버튼: [다음 →]

STEP 3 · 時  출생 시각
   질문: "태어난 시각을 아시나요?"
   입력: 시각 피커(time)  +  "시 모름" 체크
        └ '시 모름' 체크 시 시각 입력 비활성·흐리게
   도움말: "시각을 알면 더 정밀한 풀이가 가능합니다"
   버튼: [다음 →]

STEP 4 · 性  성별
   질문: "성별을 선택해 주세요"
   입력: 큰 2지선다 카드/필 (남성 / 여성) — 선택 시 골드 채움
   버튼: 선택 즉시 자동 진행 또는 [다음 →]

STEP 5 · 曆  달력
   질문: "양력인가요, 음력인가요?"
   입력: 양력 / 음력 2지선다 (음력 선택 시 '윤달' 표기 옵션 — 🟡선택)
   버튼: [다음 →]

STEP 6 · 惑  고민 (복수 선택)
   질문: "어떤 것이 궁금하세요?" (복수 선택 가능)
   입력: 칩 멀티선택 — 연애·결혼·직장·재물·건강·학업·이직·사업
        (선택 칩 골드 채움)  + (🟡 '직접 입력' 텍스트 선택)
   버튼: [다음 →]  (미선택도 통과 가능 — 선택 항목)

STEP 7 · 覽  입력 확인
   제목: "입력하신 정보를 확인해 주세요"
   내용: 입력값 요약 카드(이름/생년월일/시각/성별/달력/고민) + 각 줄 [수정] 링크(해당 단계로 점프)
   (🟡 보너스) 천간/지지 4기둥 살짝 미리보기 패널 — 나머지는 블러 + "결제 후 전체 공개"
   버튼(분기):
     - 로그인 상태 → [ 결제하러 가기 ]  (골드)
     - 비로그인   → [ 로그인하고 결제하기 ] + 안내 "결과는 로그인 후 마이페이지에서 확인"
```

### ⚙️ 백엔드 연결 (디자인은 바꾸지 말 것 — Claude는 화면만, 연결은 우리가)
- 최종 "결제하러 가기"는 아래 JSON으로 `POST /api/orders/create` → 응답 `orderId` → `/checkout/[orderId]`로 이동
- **필드 이름/타입을 절대 바꾸지 말 것**:
```json
{
  "productId": "...", "name": "", "birthDate": "YYYY-MM-DD",
  "birthTime": "HH:mm | null", "timeUnknown": false,
  "gender": "male | female", "calendar": "solar | lunar",
  "concerns": ["연애", "재물"]
}
```

### 디자인 디테일 요청
- 단계 한자 라벨(名·生·時·性·曆·惑·覽)을 골드 붓글씨로 작게
- 입력 컨트롤은 다크 위에서 잘 보이게(테두리 골드라인, 포커스 시 골드)
- 한 손 엄지 조작 고려 — 버튼은 화면 하단 고정, 큼직하게
- "다음"은 검증 통과 전 비활성(흐리게), 통과 시 골드 활성
- 뒤로 가도 입력값 유지

### Claude Design 복붙용 프롬프트 (단계별 위저드)
> 명운록(命運錄) 사주 서비스의 **풀스크린 단계별 입력 위저드**를 디자인해줘. 다크 와인(#3a0d18)+골드(#d4af6a)+명조체+한자 붓글씨 톤.
> 한 화면에 한 질문씩, 하단 골드 "다음" 버튼으로 순차 진행. 상단에 ◀이전 + 진행률 점(●○) + N/6.
> 단계: 1)이름(선택,건너뛰기 가능) 2)생년월일(한국식 날짜피커) 3)출생시각(+'시 모름' 체크) 4)성별(남/여 큰 2지선다) 5)달력(양/음력) 6)고민 칩 멀티선택(연애·결혼·직장·재물·건강·학업·이직·사업) 7)입력 확인 요약(+각 줄 수정 링크) → "결제하러 가기" 골드 버튼.
> 각 단계 fade 전환, 검증 전 다음 버튼 비활성, 뒤로 가도 값 유지. 모바일 우선, 엄지 조작 고려해 버튼 하단 고정.
> 보너스: 확인 단계에서 천간/지지 4기둥을 블러로 살짝 미리보기(전체는 결제 후). 단정·미신 표현 금지, 차분하고 고급스럽게. React + Tailwind.
