- 문서 버전: v1.0

- 작성: 혁진 / 명운봄
- 관련 산출물: `인생사주 고민퍼널 와이어프레임.dc.html`, 구현 대상 `src/components/saju/SajuWizard.tsx`

## 1. 배경 (Why)

### 현재 상태
명운봄 인생사주(종합사주)의 입력 단계는 **명식 정보(이름·생년월일·시각·성별·달력)와 고민을 한 흐름**에서 받는다.
- 고민은 칩 복수선택(`재물 / 부부·연애 / 자녀 / 직장·사업 / 건강 / 올해 운 / 노후 / 가족`) 한 단계뿐.
- 사용자가 "내 상황·내 말"로 입력할 공간이 사실상 없음 → 결과지가 일반론처럼 읽힐 위험.

### 문제
1. **이탈** — 데이터 입력(생년월일·시각)이 앞쪽에 몰려 심리적 부담이 큼. "사주 본다"는 가벼운 기대와 어긋남.
2. **개인화 연료 부족** — 고민을 카테고리로만 받으니 결과가 "내 얘기"로 느껴지지 않음. 결제 전환의 핵심 차별점(상담가 보이스)을 살릴 입력이 없음.
3. **세그먼트 부재** — 사회초년생과 은퇴 준비자에게 같은 톤·같은 깊이로 풀이됨.

### 기회 (타이트사주에서 검증된 패턴)
- 명식 데이터보다 **감정이입 질문을 먼저** 던져 가볍게 몰입시킴.
- 객관식 보기를 **공감형 한 줄**("막막해요", "지쳐요")로 작성 → 카테고리가 아니라 내 마음처럼 읽힘.
- 자유서술마다 **"안 써도 됨 + 자세할수록 더 깊게 풀어드림"** 안심문구로 부담↓·입력 품질↑.

## 2. 목표 & 성공지표 (What success looks like)

### 목표
- 입력 경험을 "설문"에서 **"상담 받으러 가는 대화"**로 전환.
- 결과지에 투입할 **사용자 자기서술 데이터**를 확보(개인화·차별화 연료).
- 인생 단계 세그먼트를 확보해 풀이 톤/우선순위를 맞춤화.

### 성공지표 (가설 — 출시 후 `/admin/analytics`로 검증)
| 지표 | 현재(기준) | 목표 |
|---|---|---|
| 입력 시작 → 무료 분석 도달률 | 측정 후 확정 | +10%p |
| 고민 자유서술 1칸 이상 작성률 | (신규) | ≥ 40% |
| 무료 분석 → 결제 전환율(CVR) | 측정 후 확정 | +15% (상대) |
| 단계별 최대 이탈 지점 | — | 특정 단계 이탈률 < 20% |

> 1인 운영 · 소규모 트래픽이므로 절대수치보다 **단계별 퍼널 드롭(상대 비교)**으로 판단.

## 3. 타깃 유저

- **주 타깃**: 30~50대, 인생 전반(돈·일·관계·건강)에 막연한 불안이 있고 "내 사주가 뭐라 하는지" 한 번 정리받고 싶은 사람.
- **공통 동기**: 점·운세에 반신반의하지만, "내 얘기를 들어주고 방향을 짚어주면" 기꺼이 소액 결제.
- **세그먼트(인생 단계)**: ①사회초년 ②한창 ③전환점 ④노후 준비 — 이후 풀이 우선순위/톤 조정에 사용.

## 4. 범위 (Scope)

### In scope
- 인생사주 단일 상품 입력 위저드 개편(고민 심화 퍼널 + 명식 정보 분리).
- 신규 입력 4단계: 인생 단계(状) / 고민 선택(惑) / 현재 상황 서술(述) / 미래 바람 서술(望).
- 확인 화면에 사용자 자기서술 재노출.
- 단계별 트래킹 이벤트 추가.
- 비로그인 → 카카오 로그인 왕복 시 입력 보존(기존 draft 메커니즘 확장).

### Out of scope (이번 아님)
- 결과지(무료/유료) 생성 로직 변경 — 단, 신규 입력 필드를 프롬프트에 전달하는 연결은 포함.
- 결제/업셀 화면 개편.
- 다국어(i18n).
- 상품 선택 단계(단일가 유지, 선택 단계 없음).

## 5. 유저플로우 (User Flow)

### 5.1 전체 퍼널
```
① 랜딩
   └─ [내 사주 무료로 확인하기]
② 카카오 로그인 (재방문 시 입력 자동 복원)
③ 고민 심화 퍼널  ★ NEW
   ├─ A. 状 인생 단계        (단일 선택 · 선택 즉시 자동 진행)
   ├─ B. 惑 고민 선택        (복수 선택 · 공감형 · 최소 1개)
   ├─ C. 述 현재 상황 서술    (자유 입력 · 선택 · 건너뛰기 가능)
   └─ D. 望 미래 바람 서술    (자유 입력 · 선택 · 건너뛰기 가능)
④ 명식 정보               (이름[선택] · 성별* · 생년월일* · 양음력* · 시각[모름 가능])
⑤ 확인                    (요약 + '내가 적은 고민' 재노출 → 수정 가능)
   └─ [무료 기본 분석 받기]
⑥ 무료 기본 분석          (명식 차트·오행 즉시 + 핵심 일부 공개, 나머지 잠금)
⑦ 결제                    (단일가 → 유료 전체 풀이)
```

### 5.2 단계별 진행 규칙
```
A 状 ── 선택 1개 ──▶ (220ms 후) 자동으로 B
B 惑 ── 1개 이상 선택 ─▶ [다음] 활성화 ─▶ C
                └─ 0개면 [다음] 비활성
C 述 ── [다음] 또는 [건너뛰기] ─▶ D
D 望 ── [다음] 또는 [건너뛰기] ─▶ ④
④ ── 필수값(성별·생년월일·달력) 충족 시 [입력 확인하기] 활성 ─▶ ⑤
⑤ ── 각 항목 [수정] → 해당 단계로 점프 후 복귀 ─▶ [무료 기본 분석 받기]
```

### 5.3 분기 / 백트래킹
- 상단 좌측 `‹` 또는 진행 점으로 **언제든 이전 단계로** 이동(입력값 보존).
- ⑤ 확인의 [수정]은 항목별 해당 단계로 직행 후 다시 ⑤로 복귀.
- 비로그인 사용자가 ⑤에서 [무료 기본 분석 받기] → **카카오 로그인 왕복**, 복귀 시 입력 전체 복원 후 이어서 진행.

### 5.4 키보드/모바일
- 모바일 퍼스트. 한 화면 = 한 질문.
- 객관식: 탭 즉시 반응. 단일선택(状)은 자동 진행, 복수선택(惑)은 [다음] 명시.
- 자유서술: 소프트키보드 고려해 [다음]/[건너뛰기]를 하단 고정.
- Enter = 다음(가능 단계 한정), 기존 위저드 동작 유지.

## 6. 기능 요구사항 (단계별 상세)

### A. 状 — 인생 단계 `lifeStage`
- 질문: "지금, 인생의 어느 길목에 계신가요?"
- 타입: 단일 선택(라디오), **선택 즉시 자동 진행**.
- 보기(값):
  - `early` 이제 막 사회에 첫발
  - `prime` 한창 일하며 달리는 중
  - `turning` 방향을 다시 잡는 전환점
  - `late` 결실 · 노후를 준비하는 때
- 필수. (자동 진행이므로 미선택 상태로 넘어갈 수 없음)
- 활용: 풀이 톤·우선순위 힌트로 프롬프트에 전달.

### B. 惑 — 고민 선택 `concerns[]` (개편)
- 질문: "요즘 가장 마음이 무거운 건 무엇인가요?" / 보조: "여러 개 골라도 괜찮아요"
- 타입: 복수 선택. **최소 1개 필수**.
- 보기(라벨 = 공감형, 값 = 영역 키):
  - `wealth` 돈 · 재물이 안 풀려요
  - `career` 일 · 진로가 막막해요
  - `relation` 사람 · 관계에 지쳐요
  - `love` 결혼 · 연애가 고민이에요
  - `family` 자녀 · 가족 문제예요
  - `health` 건강이 걱정돼요
  - `overall` 그냥 올해 전체가 궁금해요
- 안심문구: "고르신 고민을 **가장 먼저, 가장 깊게** 풀어드려요"
- 활용: 결과지 섹션 우선순위 결정.

### C. 述 — 현재 상황 서술 `situationText`
- 질문: "지금 어떤 상황인지 편하게 적어주세요" / 보조: "입력하지 않고 넘어가도 돼요"
- 타입: 멀티라인 텍스트. **선택(스킵 가능)**.
- placeholder(예시): "예) 30대 후반 직장인, 이직과 결혼을 동시에 고민 중이에요. 요즘 돈이 잘 안 모여서 마음이 무거워요."
- 안심문구(긍정 강조): "길게, 여러 개 적으실수록 **더 구체적으로** 풀어드려요"
- 제한: 최대 500자(권장), 줄바꿈 허용.
- 활용: 결과지 개인화 핵심 입력.

### D. 望 — 미래 바람 서술 `aspirationText`
- 질문: "앞으로 이루고 싶은 것이 있나요?" / 보조: "꿈 · 목표 무엇이든 좋아요"
- 타입: 멀티라인 텍스트. **선택(스킵 가능)**.
- placeholder: "예) 내 사업으로 자리잡아서 가족과 여유롭게 사는 게 꿈이에요. 건강하게 오래 일하고 싶어요."
- 안심문구: "바라는 방향을 알면 **\"언제, 어떻게\"**까지 짚어드려요"
- 제한: 최대 500자(권장).
- 활용: 시기·실행 조언 톤으로 결과지에 반영.

### ④ 명식 정보 (기존 유지, 한 화면 압축)
- `name` 이름 — 선택. 비우면 '회원님'.
- `gender` 성별 — 필수.
- `birthDate` 생년월일 — 필수.
- `calendar` 양/음력 — 필수(기본 양력).
- `birthTime` 태어난 시각 — 선택. `timeUnknown` 체크 시 시 기둥 제외.
- 카피: "마지막으로 사주 정보를 알려주세요" (감정이입을 다 받은 뒤라 부담이 낮은 위치).

### ⑤ 확인
- 요약 행: 인생 단계 / 고민 / 생년월일 / 성별·달력 — 각 항목 [수정].
- **'내가 적은 고민' 카드**: `situationText`(있으면) 요약 노출 → "내 얘기 듣고 풀어주는구나" 신뢰.
- 고지: "입력하신 정보는 명식 계산과 결과 생성에만 사용돼요".
- CTA: [무료 기본 분석 받기] (비로그인은 카카오 로그인 경유).

## 7. 데이터 모델 (FormState 확장)

```ts
type LifeStage = "early" | "prime" | "turning" | "late";

type FormState = {
  // 기존
  name: string;
  birthDate: string;
  birthTime: string;
  timeUnknown: boolean;
  gender: "male" | "female" | "";
  calendar: "solar" | "lunar" | "";
  concerns: string[];          // 영역 키 배열(개편: 라벨↔키 매핑)
  // 신규
  lifeStage: LifeStage | "";
  situationText: string;       // C. 현재 상황 (선택)
  aspirationText: string;      // D. 미래 바람 (선택)
};
```

- `payload()`에 `lifeStage / situationText / aspirationText` 추가 → `/api/orders/create` 전달 → 결과 생성 프롬프트에 반영.
- 자유서술은 trim 후 빈 문자열이면 전송 생략.

## 8. 트래킹 (analytics)

기존 `track("wizard_step", { step, total, slug })` 유지하되 단계 라벨을 확장.
- `wizard_step` — `{ step, total, slug, stepName }` (stepName: `life_stage|concern|situation|aspiration|birth|confirm`)
- `concern_select` — `{ slug, concerns: string[], count }`
- `freeform_filled` — `{ slug, field: "situation"|"aspiration", length }` (작성률 측정)
- `freeform_skipped` — `{ slug, field }`
- `checkout_login_wall` / `begin_checkout` — 기존 유지.

> 개인정보(서술 원문) 전송 금지 — 길이/작성여부만.

## 9. 카피 가이드 (명운봄 보이스)

- 상담가 톤: 단정·헤징 없이, 호칭 "회원님".
- 긍정 리프레이밍: 불안을 키우지 않고 방향을 제시.
- 금지: 가짜 카운트다운·공포 마케팅("안 보면 큰일"류).
- 객관식은 '카테고리'가 아니라 '내 마음'으로: "재물" → "돈 · 재물이 안 풀려요".
- 자유서술 안심문구는 항상 **(부담↓) + (자세할수록 이득↑)** 두 메시지를 함께.

## 10. 엣지 케이스

- 惑 0개 선택 → [다음] 비활성 + "하나만 골라도 돼요" 힌트.
- C·D 모두 스킵 → 정상 진행(결과는 명식+고민 기반으로 생성).
- 자유서술 초장문/특수문자/이모지 → 길이 제한·정상 저장(렌더 시 이스케이프).
- 음력 + 윤달 케이스 → 기존 환산 로직 따름(범위: 음력 1841~2043).
- 시각 모름 → 시 기둥 제외 명시, 결과지에서도 동일 고지.
- 로그인 왕복 중 새로고침/뒤로가기 → sessionStorage draft 복원(read-once).
- 손상된 draft → 무시하고 처음부터(기존 동작).

## 12. 오픈 이슈 / 결정 필요

1. 단계 수 증가(7 → 약 6 화면, 단 톡톡 진행) 대비 **총 입력시간 체감** — A/B로 단축형(고민 선택까지만 필수)도 검토?
2. `lifeStage`를 결과지 우선순위에 실제로 반영할지, 1차는 데이터 수집만 할지.
3. C/D 자유서술을 **하나로 합칠지**(현재+미래 한 칸) vs 2칸 유지 — 작성률 보고 결정.
